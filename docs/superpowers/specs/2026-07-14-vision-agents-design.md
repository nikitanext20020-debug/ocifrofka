# Сохранённые агенты распознавания

## Цель

Пользователь может сохранить несколько OpenAI-совместимых API-конфигураций для распознавания документов, быстро переключаться между ними и удалять ненужные. Для одного запуска распознавания используется ровно один выбранный агент.

## Хранение и миграция

Настройки остаются в `localStorage` под существующим ключом `digitizer-settings`. Конфигурация `vision` заменяется списком `visionAgents` и идентификатором `activeVisionAgentId`.

Каждый агент содержит стабильный клиентский идентификатор, имя, `baseUrl`, `apiKey` и `model`. При загрузке ранее сохранённой конфигурации с полем `vision` приложение преобразует её в первого агента и помечает его активным. Так существующие ключ и настройки не теряются.

## Интерфейс

В разделе настроек вместо единственного блока «Агент распознавания фото» показывается «Агенты распознавания». Кнопка с иконкой плюса добавляет новую конфигурацию с пустыми полями. Список содержит разворачиваемые карточки агентов.

Карточка содержит поле имени, радиокнопку выбора активного агента, поля `base_url`, API-ключа и ID модели, кнопки пресетов адресов, проверку подключения и кнопку удаления с иконкой корзины. Удаление активного агента переключает выбор на первый оставшийся. Удалить последний агент нельзя, чтобы сохранялся понятный путь к настройке распознавания.

## Поток распознавания

Экран распознавания получает активный агент из настроек и передаёт его конфигурацию в существующий `/api/extract`. Если у активного агента нет API-ключа, показано сообщение с его именем и просьбой заполнить настройки. Серверная схема и заголовки запроса не меняются: за один запрос используется одна конфигурация.

## Обработка ошибок

Проверка подключения выполняется для конфигурации конкретной карточки и показывает результат только в ней. Ошибки от API и существующая защита от локальных либо приватных `base_url` сохраняются без изменений.
