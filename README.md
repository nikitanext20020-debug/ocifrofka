# Оцифровка обращений

Веб-приложение на Next.js для распознавания заявлений по фото/PDF и переноса структурированных данных в Excel. Интерфейс и сообщения об ошибках — на русском языке.

## Возможности

- Загрузка JPG, PNG, WebP и PDF до 20 МБ; PDF рендерится в браузере через pdfjs-dist.
- Параллельное vision-распознавание с настраиваемым лимитом запросов, повтором при 429, редактируемой сессией, импортом/экспортом JSON и экспортом CSV.
- Импорт XLSX, XLS и CSV, выбор листа, быстрый постраничный просмотр больших таблиц и скачивание результата.
- ИИ-анализ колонок с поиском настоящей строки заголовков, поддержка ФИО в одной или трёх отдельных колонках, фиксированные значения категориальных полей и безопасное заполнение пропусков только в последних добавленных строках.
- Для шаблона наказов распознанный текст записывается в «Текст наказа», а «Тематика предложения» и «Направление обращения» выбираются моделью по этому тексту из значений исходной таблицы.
- Пользовательская инструкция применяется только при заполнении пропусков; строки с созданными моделью значениями явно отмечаются в выгружаемом Excel как синтетические.
- Модель возвращает только точечные изменения, которые валидируются и применяются приложением.
- История изменений и отмена последнего действия.

## Локальный запуск

Требуется Node.js 20.9 или новее.

```bash
npm install
npm run dev
```

Откройте `http://localhost:3000`, перейдите в «Настройки» и укажите ключи провайдеров либо настройте серверные ключи, как описано ниже.

## Проверка

```bash
npm run lint
npm run typecheck
npm test
npm run build
```

## Провайдеры

Приложение использует OpenAI-совместимый endpoint `chat/completions` с JSON mode. В интерфейсе доступны пресеты:

- AnyModel: `https://anymodel.org/v1/`
- RouterAI: `https://routerai.ru/api/v1`
- OpenAI: `https://api.openai.com/v1`
- DeepSeek: `https://api.deepseek.com`
- xAI: `https://api.x.ai/v1`

По умолчанию vision-агент использует `cx/gpt-5.5-review`, Excel-агент — `deepseek/deepseek-v4-flash`. В настройках распознавания можно сохранить несколько API-агентов с разными ключами и моделями, выбрать один активный перед запуском и удалить ненужные.

## Ключи и данные

Настройки и введённые пользователем API-ключи сохраняются в localStorage браузера. Сессия распознавания и очередь Excel хранятся в IndexedDB. Клиентский ключ передаётся в собственный Route Handler, используется для одного серверного запроса провайдеру и не записывается в логи. Настраиваемый `base_url` должен быть публичным HTTPS-адресом; локальные и приватные адреса блокируются.

API-ключ можно хранить только на сервере: оставьте поле ключа агента пустым и задайте переменную окружения. Для агентов распознавания используются `VISION_API_KEY_1`, `VISION_API_KEY_2` и так далее; для Excel-агентов — `TABLE_API_KEY_1`, `TABLE_API_KEY_2` и так далее. Номер соответствует позиции агента в настройках.

Чтобы сопоставление не зависело от порядка агентов, дополнительно задайте URL для того же номера:

```dotenv
VISION_API_KEY_1=secret
VISION_API_BASE_URL_1=https://anymodel.org/v1/
TABLE_API_KEY_1=secret
TABLE_API_BASE_URL_1=https://routerai.ru/api/v1
```

Сначала сервер ищет совпадение по `*_API_BASE_URL_N`, затем использует номер агента. Эти переменные не должны иметь префикс `NEXT_PUBLIC_`: тогда Next.js не включает их в клиентский JavaScript.

Изображения отправляются через серверный `/api/extract`. Таблицы целиком на сервере не сохраняются. Для табличных операций агент возвращает JSON со списком изменений, а применение выполняется в браузере.

## Деплой на Vercel

Импортируйте каталог как Next.js-проект в Vercel. Если ключи должны храниться на сервере, откройте проект в Vercel → **Settings → Environment Variables**, добавьте `VISION_API_KEY_N` / `TABLE_API_KEY_N` и при необходимости соответствующие `*_API_BASE_URL_N`, выберите нужные окружения и выполните новый деплой. Ключи из этих переменных не отправляются клиенту и не логируются.

Для крупных изображений приложение выполняет клиентское сжатие; у выбранного тарифа Vercel также должны быть достаточные лимиты тела запроса и времени выполнения функции.
